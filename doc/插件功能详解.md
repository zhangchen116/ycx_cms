# 一、插件功能详解

## 1. 整体架构

插件系统由 5 个核心部分组成：

```
┌─────────────────────────────────────────────────────┐
│                   管理后台 UI                        │

└──────────────┬──────────────────────────┬───────────┘

       ┌───────▼────────┐        ┌───────▼────────┐
```

## 2. Hook 系统（WordPress 风格）

插件系统的基石是一套轻量级的 Hook 机制，分两类：

**Action（动作）**：事件通知，无返回值。用于"在某个时机做某事"。

```19:21:cms/src/lib/hooks.ts
export function add_action(hook: string, callback: HookCallback, priority = 10) {
  _register(actions, hook, callback, priority);
}
```

```29:33:cms/src/lib/hooks.ts
export async function do_action(hook: string, ...args: any[]) {
  for (const entry of actions.get(hook) || []) {
    await entry.callback(...args);
  }
}
```

**Filter（过滤器）**：链式数据变换，有返回值。用于"修改某个数据"。

```36:41:cms/src/lib/hooks.ts
export async function apply_filters(hook: string, value: any, ...args: any[]) {
  for (const entry of filters.get(hook) || []) {
    value = await entry.callback(value, ...args);
  }
  return value;
}
```

两者都支持 `priority` 参数（数字越小越先执行，默认 10），注册时自动按优先级排序。

## 3. 可用的 Hook 清单

从 `page-renderer.ts` 的渲染流程中可以梳理出所有内置 Hook：

### Actions（动作钩子）

| Hook 名                   | 触发时机                   | 参数          | 典型用途                      |
| ------------------------- | -------------------------- | ------------- | ----------------------------- |
| `register_placeholders`   | 插件加载完成后、渲染开始前 | 无            | 注册 `data-cms-plugin` 占位符 |
| `before_render_page`      | 渲染流程最开始             | `ctx, html`   | 渲染前预处理                  |
| `after_static_binds`      | 静态绑定替换后             | `result, ctx` | 补充静态绑定逻辑              |
| `after_render_categories` | 分类列表渲染后             | `result, ctx` | 修改分类列表输出              |
| `after_render_posts`      | 文章列表渲染后             | `result, ctx` | 修改文章列表输出              |
| `after_placeholders`      | 插件占位符渲染后           | `result, ctx` | 最终修改前补充内容            |

### Filters（过滤器钩子）

| Hook 名                | 输入值             | 期望返回         | 典型用途                     |
| ---------------------- | ------------------ | ---------------- | ---------------------------- |
| `filter_page_content`  | 提取后的 body HTML | 变换后的 HTML    | 短代码替换、内容预处理       |
| `filter_rendered_html` | 最终渲染结果       | 变换后的 HTML    | 全局 HTML 后处理             |
| `wp_head`              | 空字符串           | head 区域 HTML   | 注入 `<script>`、`<meta>` 等 |
| `wp_footer`            | 空字符串           | footer 区域 HTML | 注入底部 JS 脚本             |
| `admin_menu`           | 空数组 `[]`        | 菜单项数组       | 注入后台侧边栏菜单           |
| `admin_page_<slug>`    | 空字符串           | 后台页面 HTML    | 渲染插件自定义后台页面       |

## 4. 占位符注册表

插件通过 `register_placeholder` 注册一个渲染函数，页面 HTML 中用 `data-cms-plugin` 标记位置：

```19:26:cms/src/lib/placeholder-registry.ts
export function register_placeholder(
  name: string,
  renderer: PlaceholderRenderer,
  pluginSlug: string,
  usage: string,
) {
  registry.set(name, { render: renderer, plugin: pluginSlug, usage });
}
```

渲染时，`page-renderer.ts` 会查找所有 `data-cms-plugin="xxx"` 的元素，调用对应渲染函数替换内容：

```306:329:cms/src/lib/page-renderer.ts
async function renderPlaceholders(
  html: string,
  siteSettings: Record<string, string>,
  page?: RenderContext["page"],
): Promise<string> {
  let result = html;
  const regex = /<(\w+)([^>]*\sdata-cms-plugin="([^"]+)"[^>]*)>([\s\S]*?)<\/\1>/gi;

  for (const m of result.matchAll(regex)) {
    const [fullMatch, _tag, allAttrs, pluginName] = m;
    const entry = await get_placeholder_renderer(pluginName);
    if (!entry) continue;

    const attrs: Record<string, string> = {};
    for (const attrM of allAttrs.matchAll(/data-([\w-]+)="([^"]*)"/g)) {
      attrs[attrM[1]] = attrM[2];
    }

    let pageConfig: Record<string, unknown> = {};
    if (attrs["cms-config"]) {
      try { pageConfig = JSON.parse(attrs["cms-config"]); } catch { /* ignore */ }
    }

    const mergedConfig = { ...entry.config, ...pageConfig };
    const rendered = await entry.render(attrs, mergedConfig, { siteSettings, page });
    result = result.replace(fullMatch, rendered);
  }

  return result;
}
```

**配置合并优先级**：数据库插件配置（全局） < `data-cms-config`（页面级，覆盖全局）。

## 5. 插件加载流程

```7:27:cms/src/lib/plugin-loader.ts
export async function loadPlugins() {
  if (loaded) return;    // 单例，只加载一次
  loaded = true;

  const plugins = await prisma.plugin.findMany({ where: { enabled: true } });

  for (const p of plugins) {
    const config = (p.config as Record<string, unknown>) || {};
    try {
      const mod = await import(`@/lib/plugins/${p.slug}`);  // 动态 import
      if (typeof mod.default === "function") {
        await mod.default(config);   // 调用插件的 register 函数
      }
    } catch (err) {
      console.error(`[Plugin] 加载 ${p.slug} 失败:`, err);
    }
  }

  await do_action("register_placeholders");  // 统一触发占位符注册
}
```

关键点：

- **懒加载**：插件在首次渲染页面时才加载（`loadPlugins` 被 `renderPageContent` 和 `/api/schema/page` 调用）
- **单例**：`loaded` 标志位保证整个进程只加载一次
- **数据库驱动**：只加载 `enabled: true` 的插件
- **动态 import**：路径为 `@/lib/plugins/<slug>`，即 `src/lib/plugins/<slug>/index.ts`

## 6. 内置插件分析

### contact-form（联系表单）— 占位符 + 短代码

展示了两种插入方式：

```28:46:cms/src/lib/plugins/contact-form/index.ts
export default function register(config: Record<string, unknown>) {
  // ① 注册占位符
  add_action("register_placeholders", () => {
    register_placeholder("contact-form", renderContactForm, "contact-form", '...');
  });

  // ② 注册短代码 — 页面中写 [contact-form] 也可渲染
  add_filter("filter_page_content", (html: string) => {
    return html.replace(/\[contact-form\]/g, renderContactForm({}, config));
  });
}
```

- 占位符方式：`<div data-cms-plugin="contact-form"></div>`
- 短代码方式：`[contact-form]`（通过 filter 替换）

### analytics（统计代码）— wp_head 注入

不注册占位符，纯 filter 插件：

```28:38:cms/src/lib/plugins/analytics/index.ts
export default function register(config: Record<string, unknown>) {
  const provider = config.provider || "ga";
  const id = config.id as string;

  if (!id) return; // 未配置 ID 则不注入

  add_filter("wp_head", (head: string) => {
    if (provider === "baidu") return head + baiduScript(id);
    return head + gaScript(id);
  });
}
```

配置通过后台 JSON 编辑器设置（如 `{ "provider": "ga", "id": "G-XXXXXX" }`）。

### search（站内搜索）— 占位符 + wp_footer + 独立 API

最复杂的内置插件，展示了完整模式：

1. 注册 `search` 占位符（渲染搜索框 HTML）
2. 通过 `wp_footer` 注入前端搜索 JS
3. 调用独立的 `/api/search` 公开端点

```98:121:cms/src/lib/plugins/search/index.ts
export default function register(config: Record<string, unknown>) {
```

## 7. 后台菜单注入

插件可以通过 `admin_menu` filter 向后台侧边栏注入菜单项：

```5:9:cms/src/app/api/admin/menu/route.ts
export async function GET() {
  await loadPlugins();
  const menuItems = await apply_filters("admin_menu", []);
  return NextResponse.json(Array.isArray(menuItems) ? menuItems : []);
}
```

菜单项格式为 `{ slug: string; label: string; icon?: string }`，点击后访问 `/admin/plugin/<slug>`，该页面通过 `admin_page_<slug>` filter 获取内容：

```5:13:cms/src/app/api/admin/plugin-page/[slug]/route.ts
export async function GET(
  _req: Request,
  { params }: { params: Promise<{ slug: string }> },
) {
  const { slug } = await params;
  await loadPlugins();
  const content = await apply_filters(`admin_page_${slug}`, "");
  return NextResponse.json({ content });
}
```

## 8. 插件安装（zip 上传）

后台支持上传 zip 包安装插件。`/api/plugins/upload` 的逻辑：

1. 接收 zip 文件，解压到 `src/lib/plugins/` 目录
2. 读取 `plugin.json` 获取元数据
3. Upsert 数据库 `Plugin` 记录（默认 `enabled: true`）

zip 包结构支持两种：

- 扁平结构：`index.ts` + `plugin.json` 在根目录
- 目录结构：`<slug>/index.ts` + `<slug>/plugin.json`

## 9. 数据库模型

```193:205:cms/prisma/schema.prisma

  id          String   @id @default(cuid())
```

## 目录结构

```
src/lib/plugins/<your-slug>/
├── index.ts          # 必须有，default export register 函数
├── plugin.json       # 推荐，元数据（zip 安装时读取）
└── ...其他文件        # 可选，辅助模块
```

## plugin.json 规范

```json
{
  "name": "插件显示名",
  "description": "插件描述",
  "version": "1.0.0",
  "author": "作者名",
  "config": {
    "key": "default-value"
  },
  "hooks": [
    { "hook": "register_placeholders", "type": "action" },
    { "hook": "wp_head", "type": "filter" }
  ]
}
```

- `config`：默认配置，安装时写入数据库，后台可修改
- `hooks`：声明使用的 hook（仅用于后台展示，不影响实际执行）

## 完整插件示例：评论系统

```typescript
// src/lib/plugins/comments/index.ts
import { add_action, add_filter } from "@/lib/hooks";
import { register_placeholder } from "@/lib/placeholder-registry";
import { prisma } from "@/lib/prisma";

// 前端评论组件 JS
const COMMENTS_JS = `
(function() {
  document.querySelectorAll('.cms-plugin-comments').forEach(function(container) {
    var pageId = container.dataset.pageId;
    var list = container.querySelector('.comment-list');
    var form = container.querySelector('.comment-form');

    function load() {
      fetch('/api/plugins/comments/list?pageId=' + pageId)
        .then(function(r) { return r.json(); })
        .then(function(data) {
          list.innerHTML = data.comments.map(function(c) {
            return '<div class="py-2 border-b"><strong>' + c.name + '</strong>'
              + '<p class="text-sm text-gray-600 mt-1">' + c.content + '</p></div>';
          }).join('') || '<p class="text-gray-400 text-sm">暂无评论</p>';
        });
    }

    form.addEventListener('submit', function(e) {
      e.preventDefault();
      var fd = new FormData(form);
      fd.append('pageId', pageId);
      fetch('/api/plugins/comments/submit', { method: 'POST', body: fd })
        .then(function() { form.reset(); load(); });
    });

    load();
  });
})();
`;

function renderComments(
  attrs: Record<string, string>,
  config: Record<string, unknown>,
  context: { siteSettings: Record<string, string>; page?: { id: string } }
): string {
  const pageId = attrs["page-id"] || context.page?.id || "";
  const title = (config.title as string) || "评论";

  return `
<div class="cms-plugin cms-plugin-comments my-8" data-page-id="${pageId}">
  <h3 class="text-lg font-bold mb-4">${title}</h3>
  <div class="comment-list mb-4"></div>
  <form class="comment-form space-y-2">
    <input name="name" placeholder="昵称" class="w-full border rounded px-3 py-2 text-sm" required />
    <textarea name="content" placeholder="说点什么..." rows="3"
      class="w-full border rounded px-3 py-2 text-sm" required></textarea>
    <button type="submit" class="bg-blue-600 text-white px-4 py-2 rounded text-sm">发表评论</button>
  </form>
</div>`;
}

export default function register(config: Record<string, unknown>) {
  // ① 注册占位符
  add_action("register_placeholders", () => {
    register_placeholder("comments", renderComments, "comments",
      '在页面中插入评论区。\n\n' +
      '```html\n<div data-cms-plugin="comments"></div>\n```\n\n' +
      '可指定 page-id 绑定到特定页面：\n' +
      '```html\n<div data-cms-plugin="comments" data-page-id="xxx"></div>\n```'
    );
  });

  // ② 注入前端脚本
  add_filter("wp_footer", (footer: string) => {
    return footer + `<script>${COMMENTS_JS}</script>`;
  });

  // ③ 注入后台菜单（可选）
  add_filter("admin_menu", (menu: Array<{ slug: string; label: string }>) => {
    return [...menu, { slug: "comments", label: "评论管理" }];
  });

  // ④ 渲染后台管理页面（可选）
  add_filter("admin_page_comments", async () => {
    // 这里可以查询数据库返回管理界面 HTML
    return "<div>评论管理页面（示例）</div>";
  });
}
```

```json
  "config": {
```


## 调试技巧

1. **查看加载日志**：插件加载失败会在服务端控制台输出 `[Plugin] 加载 xxx 失败`
2. **确认插件启用**：后台 `/admin/settings` 检查插件开关状态
3. **验证 Hook 注册**：访问 `/api/schema/page` 可看到已注册的占位符列表
4. **配置生效**：后台编辑 JSON 配置后，需要重新渲染页面才会生效（因为 `loadPlugins` 是单例，开发时需重启服务）

## 总结

这个插件系统借鉴了 WordPress 的成熟设计，但在现代 TypeScript/Next.js 环境下重新实现。它的核心价值在于：

1. **统一扩展点**：通过 Hook + 占位符，插件可以介入渲染流程的任意环节
2. **数据库驱动**：开关、配置都存数据库，无需改代码即可启停
3. **AI 可见**：占位符的 `usage` 说明会注入到 `/api/schema/page`，AI Agent 能自动发现可用插件
4. **热安装**：zip 上传即装即用，适合分发
